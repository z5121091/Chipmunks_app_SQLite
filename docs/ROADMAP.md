## 一、PDA扫描模式适配

### 功能概述
支持PDA硬件扫描枪，提供独立的扫描界面，实现毫秒级快速扫描。

### 实现方案
**方案C：智能记住 + 快速切换**

- 首次使用：弹窗选择默认扫描模式，勾选"记住选择"
- 后续启动：自动进入上次选择的扫描界面
- 切换方式：扫描页面顶部模式指示器，点击可切换
- 设置页面：可修改默认扫描模式

### 界面设计

**相机模式**：

├─────────────────────────────────────┤
│      ┌─────────────────────┐       │
│      │    相机预览区域      │       │
│      └─────────────────────┘       │
├─────────────────────────────────────┤
│ 最近：GH20N40G | 500个 | 刚刚      │
└─────────────────────────────────────┘
```

**PDA模式**：
```
┌─────────────────────────────────────┐
│ 扫描          [📷 相机 | 🔫 PDA]    │

│ 订单：ORD-001    [切换]             │
├─────────────────────────────────────┤
│      ┌─────────────────────┐       │
│      │   ⏳ 等待扫描...     │       │

### 技术要点

1. **隐藏输入框接收PDA输入**
```tsx
<TextInput
  style={{ width: 0, height: 0, opacity: 0 }}
  autoFocus={true}
  showSoftInputOnFocus={false}
  onChangeText={handleScanInput}
  onSubmitEditing={handleScanComplete}
/>
```

2. **扫描事件识别**
- PDA扫描通常以回车结束
- 需要识别扫描速度区分手动输入

3. **反馈机制**
| 状态 | 声音 | 震动 | 视觉 |
|------|------|------|------|
| 成功 | 清脆滴声 | 短震 | 绿色勾 |
| 重复 | 警告音 | 连续震动 | 红色警告 |
| 错误 | 错误音 | 长震 | 红色提示 |

## 二、入库统计功能

### 功能概述
快速扫描统计到货物料，按型号自动汇总数量，支持验货核对。

### 使用场景
```
供应商送货 → 30箱货 → 多个型号
     ↓
PDA扫描每一箱
     ↓
自动汇总结果：
  GH20N40G：10箱，共5000个
  GH30N60：8箱，共4000个
  GH35N60：12箱，共6000个
     ↓
导出报表 / 确认入库
```

### 与现有功能对比

| 对比项 | 现有扫描入库 | 入库统计 |
|--------|-------------|---------|
| 目的 | 记录每箱物料信息 | 快速统计到货数量 |
| 粒度 | 每箱单独记录 | 按型号汇总统计 |
| 订单 | 必须绑定订单 | 可独立于订单 |
| 输出 | 详细物料清单 | 汇总报表 |

### 界面设计
```
┌─────────────────────────────────────────────┐
│  📊 入库统计                         [清空] │
├─────────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐   │
│  │      ⏳ 等待扫描...                 │   │
│  │      已扫描：23 箱                  │   │
│  │      型号数：5 种                   │   │
│  └─────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│  汇总结果                                   │
│  ┌─────────────────────────────────────┐   │
│  │ GH20N40G      10箱   5,000个   ▼   │   │
│  │ GH30N60       8箱    4,000个   ▼   │   │
│  │ ──────────────────────────────────  │   │
│  │ 合计          23箱   11,500个       │   │
│  └─────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│  [继续扫描]  [导出报表]  [确认入库]         │
└─────────────────────────────────────────────┘
```

### 功能流程
1. 进入"入库统计"页面
2. PDA扫描，实时汇总
3. 扫完确认
4. 选择操作：导出报表 / 确认入库 / 清空重来

### 确认入库选项
- 绑定现有订单
- 创建新订单
- 仅保存统计记录

## 三、ERP系统联动

### 功能概述
与ERP系统对接，实现采购订单推送、入库回传、库存同步等业务闭环。

### 联动价值
```
ERP系统                    掌上仓库APP
┌─────────┐                ┌─────────┐
│ 采购订单 │ ──推送──→     │ 待入库单 │
└─────────┘                └─────────┘
                                ↓
                           PDA扫描入库
                                ↓
┌─────────┐                ┌─────────┐
│ 库存更新 │ ←──回传──     │ 实收数据 │
└─────────┘                └─────────┘
```

### 典型业务流程
1. **ERP创建采购订单** → 自动推送到APP
2. **APP收到待入库通知** → 订单详情一目了然
3. **仓库PDA扫描验收** → 扫描统计核对
4. **APP确认入库** → 自动回传实收数据
5. **ERP自动更新** → 库存增加、订单状态更新、生成入库单

### 联动方案

#### 方案一：API对接（推荐）
```
ERP ←─ API调用 ─→ Express Server ←─→ APP

接口设计：
ERP → APP：
- POST /api/v1/erp/purchase-orders    推送采购单
- PUT  /api/v1/erp/orders/{id}/cancel 撤销订单

APP → ERP：
- POST /erp/api/inbound               入库回传
- POST /erp/api/inventory/check       库存查询
```

#### 方案二：中间数据库
```
ERP ←─→ 中间库(MySQL) ←─→ APP服务端
适合：ERP不支持API、需要解耦
```

#### 方案三：文件交换
```
ERP导出 ──→ 文件交换 ──→ APP导入
适合：老旧ERP、非实时场景
```

### 可联动功能点

| ERP功能 | APP联动 | 价值 |
|---------|---------|------|
| 采购订单 | 推送到APP待入库 | 不用手动创建订单 |
| 销售订单 | 推送到APP待发货 | 发货时扫码核对 |
| 库存查询 | APP实时查询库存 | 现场查库存 |
| 入库单 | 自动生成入库单 | 无需二次录入 |
| 出库单 | 扫码生成出库单 | 溯源有据可查 |
| 物料主数据 | 同步物料信息 | 保证数据一致 |

### 实施步骤
1. **调研**：了解现有ERP系统类型、接口能力
2. **设计**：确定对接方案、接口规范
3. **开发**：服务端API开发、数据映射
4. **测试**：联调测试、异常处理
5. **上线**：灰度发布、监控优化

### 前置条件
- 明确ERP系统类型（金蝶/用友/ SAP/自研等）
- ERP系统具备API能力或开放数据库访问
- 业务流程梳理确认

### 优先级
**高** - 核心业务价值，提升整体效率
